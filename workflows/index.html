Overview

Workflow represents a set of rules and statuses which specify the life cycle of a case. A typical workflow defines all possible actions and transitions through statuses for all cases.

To create a new Workflow go to Setup >> Workflows >> Add Workflow.

Step 1: Enter Workflow info

At this step you will see three fields that should be filled in order to create a new Workflow. That fields are: Name, Description and Unique Key.

  • Unique Key identifies a Workflow and it can’t be changed. It can’t contain spaces and it must be in ASCII encoding, no UTF8 characters are allowed.

Step 2: Add Statuses

At this step you have to select all status that you want to have in your new Workflow through which you case will go during their life cycles. You can choose Available Statuses or Add new Status.

  • Statuses are dragged from the left to the right part of the page via drag & drop mechanism.
  • At least one status must be selected. You can add and remove statuses in any time once a workflow is created.
  • If you want to search for a status try typing in Search box, or click on search options. Click here to learn more.

Step 3: Form Designer

At this step you can design layout of the fields which user will see when creating a new case of this Workflow type, by adding Custom Fields to this page via drag & drop. You can choose Custom Field(s) from existing Custom Fields or you can Add New if you need it for this Workflow.

Overview

  • A status is the state or position a case is in at a certain point in time.
  • Statuses represent the progression of a case through its lifecycle from open to closed. A case can only be in one status at a given time.
  • Statuses can be associated to any number of Workflows. A workflow can have an unlimited amount of statuses.
  • Only administrative users can add and manage statuses, but registered users can view the statuses of a case and see what status a case is on by going to the case page.

To access Statuses Page go to Setup >> Workflows >> Statuses.

Create a New Status

  1. Click New Status
  2. Fill out the form fields – Name and System ID. And choose one of the following actions to your new status, and choose in which workflows your status will be included.
    • System ID will automatically populate as you name the department.
  3. Click Add
  4. Your status will appear in the table of listed statuses.

List of existing Statuses

Here you can see list of already created statuses. Information about statuses which you can see is Name of the status, System ID, and you can execute some actions under the certain status.

  • To edit status click on Actions button near desired status, and then on Edit.
  • If you want to see when and by whom the status is created click on Actions button near desired status, and then on Details.
  • If you want to translate name of the status, click on Actions button near desired status, and then on Translate. Choose dictionary and write translation.
  • If you want to export data for backup purposes or to import to another tenant, click on Actions button and then on Download Data. It will download an .wfb file with your object exported in Jason format. You can import it via Migration Manager to another tenant.
  • To search for a status try typing in Search box, or click on search options. Click here to learn more.

Overview

WorkFlow is a web-based application designed for managing business processes. It includes everything you need to organize, execute, and analyze business processes to connect people, data, and daily activities.

  • Workflow allows users to automate various processes through the use of projects, workflows, and cases.
  • There are three types of roles within the WorkFlow application: administrator, user and guest:
    • Administrators have the ability to create users, set permissions, and create customizations to the WorkFlow application.
    • Users are able to create cases and perform actions on a case to change its status from opened to closed.
    • A Guest is someone who does not have a username or password to log into Workflow but they can create and view their own cases.

To access Workflow Page go to Setup >> Workflows >> Workflows.

Fundamentals of Workflow

Workflows allows users to automate various processes through the use of projects, workflows, and cases.

  • Project is a mandatory way of grouping cases. Think of it as grouping cases into different categories. For example, if you were to use workflow as a method of tracking the tasks associated with onboarding new employees, which is a common use case for Workflow, a project might be Employee Management.
  • Workflow represents a specific business process and is comprised of a series of steps which the user needs to take in order to complete the represented business process. These steps are called statuses, and the user performs actions to move from one status to the next.
  • Case travels through each status of a workflow. For example, in our employee onboarding workflow, a user in the Human Resource department could log into Workflow and see a case for a new employee. That user could then perform certain actions to move the case along from status to status.

Create a New Workflow

If you want to create a new Workflow, click on the ‘New Workflow’ link and a wizard for creating a new Worflow will open. See Add New Workflow.

List of existing Workflows

You can see list of already created workflows in the Workflows tab. Information about workflows which you can see is Name of the workflows, Description, Unique Key, and you can execute some actions under the certain workflow.

  • To edit workflow click on Actions button near desired workflow, and then on Edit.
  • If you want to see when and by whom the workflow is created click on Actions button near desired workflow, and then on Details.
  • If you want to modify workflow, click on Actions button near desired workflow, and then on Design View. It opens a workflow designer, and this option is available only in Professional Edition of Workflow.
  • If you want to access the table view of the workflow, click on Actions button near desired workflow, and then on Table View. There is short explanation of the page, and two tabs. First tab opens by default when entering into Table view of the matrix and it displays all actions of that workflow in a transition matrix. Second tab is used for settings for defaults actions in this workflow.
  • If you want to access the list view of the workflow, click on Actions button near desired workflow, and then on List view. In the list view of the workflow there are few sections on the left side of the page and the main part of the page shows table with listed actions.
  • To change statuses for workflow, click on Actions button near desired workflow, and then on Statuses. A popup window in which you can add or remove statuses from this workflow will open.
  • If you want to translate name of the workflow, click on Actions button near desired workflow, and then on Translate. Choose dictionary and write translation.
  • If you want to export data for backup purposes or to import to another tenant, click on Actions button and then on Download Data. It will download an .wfb file with your object exported in Jason format. You can import it via Migration Manager to another tenant.
  • To delete workflow, click on Actions button an then on Delete. The Delete action can’t be executed when:
    • There is at least one case that belongs to choosen workflow.
  • If you want to delete all cases and workflows that you’ve created, click on Actions button an then on Delete Workflow and all cases.
  • To search for a workflow try typing in Search box, or click on search options. Click here to learn more.

The Default Workflow

In the Default Workflow tab you can choose one of your created workflows to become the default workflow for users.
