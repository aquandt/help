Overview

Departments are objects that help organize users. Department in Workflow represents a real department in the company.

Usage of departments

  • Departments group the users that belong to a company’s department within a company, and execution and visibility permissions can be set around departments
  • Departments are used for setting user visibility by defining who will be able to see users of a department. When a permission is selected for a user visibility permission that means that a group of users (defined in selected permission groups) can see all users from this department.
  • To “be able to see a user” means that someone can:
    • access that user’s profile when clicking on the user name anywhere in application
    • see that user in the Assign to fields (if admin haven’t restricted it on a particular action)
  • Departments and User Types are used for creating Permission Groups. Permission groups are used for setting workflow, project, user, contact, account and action visbility permissions
  • You can use departments in setting up notifications groups

More about departments

  • Departments may belong to a Company, but it is not mandatory
  • Departments may represent real departments in your company, but also can be some other logical group of people that is convenient for setting permissions
  • You can have any number of users in a department - zero, one or more

To access Department Page go to Setup >> Users >> Departments.

Create a New Department

  1. Click New Department
  2. Fill out the form fields – Name, System ID, Gate Keeper, Company, System Roles and Permission Group
    • System ID will automatically populate as you name the department.
    • Gate Keeper is the user who is responsible for the department. A gate keeper can be automatically assigned cases, or have permissions that other users in the department do not have. This role can also be used in defining to whom the notifications will be send.
    • Adding Companies and attaching users to a company is an optional feature. Multiple departments can be included in a company. If you are using companies, the company of the departments can be specified here.O
    • With System Roles and Permission Groups you can set which users are visible to users from this department.
  3. Click Submit
  4. Your department will appear in the table of listed departments.

List of existing Departments

You can see list of already created departments in the Department tab. Information about department which you can see is Name of the department, Company that belongs, Gate Keeper of the department, and you can execute some actions under the certain department.

  • To edit department click on Actions button near desired department, and then on Edit.
  • If you want to see when and by whom the department is created click on Actions button near desired department, and then on Details.
  • To see users who belong to the desired department, click on Actions button and then on Show Users.
  • If you want to translate name of the department, click on Actions button near desired department, and then on Translate. Choose dictionary and write translation.
  • If you want to export data for backup purposes or to import to another tenant, click on Actions button and then on Download Data. It will download an .wfb file with your object exported in Jason format. You can import it via Migration Manager to another tenant.
  • To delete department, click on Actions button an then on Delete. The Delete action can’t be executed when:
    • The desired department is the Default department
    • The desired department is already in use
  • To search for a department try typing in Search box, or click on search options. Click here to learn more.

The Default Department


In the Default Department tab you can choose one of your created departments to become the default department for users. Because the department is optional when creating a new user, this default department will be assigned to users that do not have it selected.
