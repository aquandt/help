Overview

Company is an optional object which administrators may enable to have an additional level or grouping. It acts as a container to group departments and can be used to create a relationship between the department and project.

  • Departments in Workflow may belong to a company, but also department can exist without belonging to any company.
  • Companies can be used if you want to group users by companies, and then by departments and user types.
  • Workflow application can have public and internal comments and attachments. Those “internal” comment and attachments are visible only to people who belong to the same Company. For example, if all your users in reality belong to one company, but you want different teams to have ability to have internal comments/attachments, then you need to create a company (and at least one department in that company) for each team.

Create a New Company

  1. Click New Company
  2. Fill out the form fields
    • Company code you can use in integration with CPQ if is needed
    • Corresponding projects are used when you have a case but it doesn’t belong to any project, so it will be placed in Corresponding project of case creator’s company. If corresponding project is not populated, tenant’s default project will be used.
  3. Click Add
  4. Your company will appear in the table of listed companies.

List of existing Company

Here you can see list of already created companies. Information about companies which you can see is Name of the company, Address, State, Country, and you can execute some actions under the certain company.

  • To edit company click on Actions button near desired company, and then on Edit.
  • If you want to see when and by whom the company is created click on Actions button near desired company, and then on Details.
  • To see departments who belong to the desired company, click on Actions button and then on Show Departments.
  • If you want to export data for backup purposes or to import to another tenant, click on Actions button and then on Download Data. It will download an .wfb file with your object exported in Jason format. You can import it via Migration Manager to another tenant.
  • To delete company, click on Actions button an then on Delete. The Delete action can’t be executed when: *There is a department in that company
  • To search for a company try typing in Search box, or click on search options. Click here to learn more.
