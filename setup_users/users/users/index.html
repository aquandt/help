Overview

User is a person who uses WorkFlow. They can be organized into departments and user types, and have assigned permisssions in the system to which determine what they can see and what actions they can perform.

  • You can set to which user type it belonng and in which department is
  • Every Permission Group, Assignment Group, User Type and Department, are formed of users
  • You can assign case to specific user
  • When you edit workflow actions, you can set which user(s) can execute that action, to whom the notifications will be sent and to whom the case will be assigned
  • You can set which users will be administrator(s). Admins are users which can access Admin page (Setup) and create and customize workflows, make cases, create new users, departments, permission groups, assignment groups, company..
  • Users can be used to adjust notifications
  • Project roles can be assigned to users
  • Each user belongs to exactly one department and exactly one user type. If you don’t select any department or user type when you create a user, default ones will be used.
  • There is no hierarchy of users.
  • User can change some settings for himself/herself by clicking on his/hers name and choosing User Settings (Information about user, Change Password, set Time and Date Format and set rules for assigning Notifications), or View Profile (Basic Information, Recent Activities, Team, Projects, Out of Office)
  • Users can see profiles of other users by clicking of the name of the user displayed somewhere in application (on Users page, in cases or lists of cases as creators, updated by, assignee, case team, etc).
  • User visibility, or which users are visible to which users is defined by Department.

To access User Page go to Setup >> Users >> Users.

Create a New User

  1. Click New User
  2. Fill out the form fields
    • When you’re creating a new user, you can set to which department and which user type your new user will
    • If Administrator checkbox is selected, user will be admin and will have access to application Setup
    • Federation ID is only used if your tenant is using Federation for Single Sign On (SSO). To learn more about SSO possibilities of Workflow application, click here.
  3. Click Add
  4. Your user will appear in the table of listed user

List of existing Users

Here you can see list of already created users. Information about users which you can see is User’s Name, Email, Department, User Type, Last Login, and you can execute some actions under the certain user.

  • To edit user click on Actions button near desired user, and then on Edit.
  • If you want to see when and by whom the user is created click on Actions button near desired user, and then on Details.
  • To set profile photo, click on Actions button and then on Upload Photo.
  • If you want to export data for backup purposes or to import to another tenant, click on Actions button and then on Download Data. It will download a .wfb file with your object exported in Jason format. You can import it via Migration Manager to another tenant.
  • To view Permission Details, click on Action button near desired user, and then on View Permission Details. When you click on it, a pop up screen will display with additional tabs:
    • The Workflows tab you can see any workflows and access rights to those workflows for the user. These permissions cannot be modified from the workflows tab.
    • The Permission Groups tab shows all permission groups to which the user belongs.
    • The Projects tab shows all projects within this environment or tenant which the user has permission to view.
    • The Users tab shows all WorkFlow users which this person can view.
    • The Custom Fields tab displays a table listing of all administrator-created fields which this user can see.
  • To Delete user, click on Actions button and then on Delete. User can’t be deleted if user:
    • Performed any action on any case
    • There’s a case where your user is assignee
    • User was admin and user created or modified any entity in Setup!
    • User settings – user excluded for delete
  • To search for a user try typing in Search box, or click on search options. Click here to learn more.

The User Custom Fields

To access User Custom Fields go to Setup >> Users >> Users >> User Custom Fields.

You can use User Custom Fields if you want to have some additional information about user. These custom fields are visible on user’s profile page. You have Custom, Standard fields, Forms and UI.

In the User Custom Fields tab you can set permissions about custom fields. You can:

  • Select which users will be able to see user custom fields. Clicking on this link opens a popup where you can select system roles and permission groups which will have ability to edit user custom fields.
  • Select which users will be able to edit user custom fields. Clicking on this link opens a popup where you can select system roles and permission groups which will have ability to edit user custom fields.
  • Add custom fields to user. This opens Form Designer where you can add all custom fields you need.

The Public Sign Up

If you want to allow people to create users in your tenant by themselves, you should be using Public Sign Up. When you choose Department and User type and click on the button Generate Public SignUp Link, application will generate a link to you. This link should be embedded somewhere on your website. Clicking on it will open a page which a person can populate and when it is submitted, a new user will be created. This new user will be placed in department and user type which you selected when you generated Public Signup Link.
