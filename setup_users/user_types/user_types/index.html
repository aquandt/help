Overview

User Types represent group of users with similarities or group of users who play the same role within a company.

  • If you want to join users who have the same role or doing the same activities, you should set that all of them are the same user type
  • User Types and Departments are used for creating Permission Groups. Permission groups are used for setting workflow, project, user, contact, account and action visbility permissions
  • Every user belongs to a user group which helps define characteristics for similar users
  • If you don’t have many users, or if you don’t need to classify users by two different ways, place all users in one user type, and use departments for user classification

To access User Page go to Setup >> Users >> User Types.

Create a New User Type

If you want to create a new user type you just have to click on the New User Type. You will see fields Name and System ID which you have to fill in order to create a new user type. The System ID field will automatically populate as you name the user type. New user type that you’ve created will appear in the table of listed user types.

List of existing User Types

You can see list of already created user types in the User Types tab. Here you can execute some actions under the certain user type.

  • To edit user type click on Actions button near desired user type, and then on Edit.
  • If you want to see when and by whom the user type is created click on Actions button near desired user type, and then on Details.
  • To see users who belong to the user type, click on Actions button and then on Show Users.
  • If you want to translate name of the user type, click on Actions button near desired user type, and then on Translate. Choose dictionary and write translation.
  • If you want to export data for backup purposes or to import to another tenant, click on Actions button and then on Download Data. It will download an .wfb file with your object exported in Jason format. You can import it via Migration Manager to another tenant.
  • To delete use type, click on Actions button an then on Delete. The Delete action can’t be executed when:
    • The desired user type is the Default user type
    • There are users who belong to this user type
  • To search for a user type try typing in Search box, or click on search options. Click here to learn more.

The Default User Type

You can set the default user type in the Default User Type tab. From here, you can choose one of your created user types to become the default user type for users. Because the user type is optional when creating a new user, this default user type will be assigned to users that do not have it selected.
