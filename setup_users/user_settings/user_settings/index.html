Overview

User Settings represent page where you can set some caracteristics for all users, but itâ€™ll applies only to current instance (current tenant or current standalone installation) of Workflow in which they are entered.

To access User Settings Page go to Setup >> Users >> User Settings.

When the page opens you will see several fields that should be filled in order to customize User Settings. Those fields are about email address, date and time format and password.

Emails

Here you can see two checkboxes:

  • Users can have duplicate emails - you can set wheter you want for user to have one email address or he can have duplicate.
  • User must have an email address - if this option is checked you won’t be able to add new user without his email adress.

User can set default date, time and timezone format.

Password Management

In Password Management section user can define rules for new users:

  • Password must contain: You can choose type of characters that password must contain
  • Password expires after X days: If value is 0, user password won’t expire. For other values after X days, he won’t be able to login application.
  • Last four passwords may not be reused: If this option is checked, user won’t be able to use password same as the four last passwords.
  • Force user to change password on next login: If this option is checked, every new created user will be forced to change his password on first login.
  • Password can’t be shorter than X character(s): If this option is checked, password must have at least X character(s).
  • Password can not be too similar to first, last or user name: If this option is checked, password can’t contain first, last or user name.
  • Users excluded from password rules: Choose users that will be excluded from these password default rules.
