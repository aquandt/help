Home page

Users

User can be organized into departments and user types, and have assigned permisssions in the system to which determine what they can see and what actions they can perform.

Departments

Departments are objects that help organize users. Department in Workflow represents a real department in the company.

User Types

User Types represent group of users with similarities or group of users who play the same role within a company.

Companies

Company is an optional object which administrators may enable to have an additional level or grouping.

Assignment Groups

Assignment Group represents a set of users defined by selecting the combination of system permissions, permission groups, project roles and specific users.

Workflows

WorkFlow is a web-based application designed for managing business processes. It includes everything you need to organize, execute, and analyze business processes to connect people, data, and daily activities.

Add Workflow

Workflow represents a set of rules and statuses which specify the life cycle of a case. A typical workflow defines all possible actions and transitions through statuses for all cases.

Statuses

Statuses represent the progression of a case through its lifecycle from open to closed. A case can only be in one status at a given time.

Projects

Each case belongs to a project. Project can contain any number of cases. Project is used to group similar cases, a holder for cases.
